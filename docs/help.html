RISQUE help

Quick reference for this app — read before you log in or share with new players.

Login screen

  • Player rows — Enter a name, then click a color swatch to assign that color. Each name and each color must be unique.
  • Game manual — Full rules overview (Risk-style).
  • Help — This page.
  • Log in — Starts a new game with the players you entered (after setup phases).
  • Load game — Choose a saved .json game file to restore a previous session.

Where the game runs

Most play happens in game.html (runtime shell): the map, phase panels, and buttons for attack, deploy, income, card play, etc. The exact screen depends on the current phase saved in your game state.

Map & attack phase (typical)

  • Click one of your territories with at least 2 armies, then an adjacent enemy territory (or valid aerial link if enabled).
  • Use Roll, Blitz, or other attack buttons on the panel.
  • After capturing, you will be prompted to move armies into the conquered territory (at least the minimum required).
  • Reinforcement phase moves you to placing armies between your territories before drawing cards / ending the turn, when the flow calls for it.

Saves & browser data

  • The active game is kept in the browser’s localStorage under the key used by the app (e.g. when you continue from the runtime).
  • Save game on the map suggests a name like RISQUE 2026-04-17 8:11AM (two spaces after RISQUE and before the time). The colon in the time becomes a hyphen in the saved file on Windows (e.g. 8-11AM). If your browser supports it (often with http://localhost or HTTPS), the Save dialog opens in Downloads by default and you can change the folder. As a local file:// page, the file name is prompted and the file usually goes to your Downloads folder.
  • Load game on login replaces that stored state from your JSON file.
  • Clearing site data for this origin will erase in-progress games unless you have exported a JSON backup.

Privacy gate

When the app shows a privacy step, it is so the next player does not see secret information (hands, draws, etc.). Pass the device or switch display as your group prefers.

Instant replay

When the game has stored a replay tape for the current session, the host sees a REPLAY control (in the HUD title area) on supported screens, including postgame. Starting replay:

  • Opens a short start dialog: full replay from the territory deal (default), skip to the first battle, or (for newer recordings) jump to the first frame at or after a chosen round. Then it plays the recorded map in order: territory deal (random assignment, one step per territory), setup deploy steps, battle outcomes, plus elimination splashes when someone is removed.
  • Shows a bar labeled INSTANT REPLAY with SKIP TO END and CANCEL. Press Esc to cancel replay.
  • Does not re-roll dice or flip cards; it advances stored board snapshots so everyone can recall how the map evolved.
  • A public / TV tab that mirrors the host should receive the same frames if mirroring is set up for normal play (same origin / storage as the host).

Replay is recorded only on the host session. If the button stays disabled, the save may predate replay data or the session may not have recorded an opening yet (e.g. no deal/deploy/battle segments on tape). Older tapes may not include per-frame round stamps, so the “from round” option can be unavailable until you record a new session with an updated build.

Public monitor — cardplay recap

During committed cardplay on the public display, when a wildcard is used as a territory card, the monitor shows the chosen territory name prominently; a second line beneath indicates the effect at a glance (for example +2, −2, or ACQUIRED), similar to normal territory cards.

If something looks wrong

  • Combat dice or income once looked wrong — Save, refresh, and continue; if it persists, load a backup JSON. Odd totals can happen if the browser had a stale tab or interrupted save.
  • Refresh the page; if the save is corrupt, try Load game from a known-good JSON file.
  • Use the launcher / tools menu from index.html?menu=1 if your project includes it (clear logs, test setups).
  • For a fresh start, use login again with new players (or load an earlier save).
Tip: Open Manual and Help in a separate tab from login so you can read while keeping the login form open.
